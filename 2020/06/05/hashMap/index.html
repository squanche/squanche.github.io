HashMap

概念
名称 说明
length Table数组的长度
size 成功通过 put 方法添加到HashMap 中的所有元素的个数
hashCode object.hashCode()返回的值，尽可能离散均匀分布
Hash Object.HashCode()与当前集合的 table.Length(进行位运算的结果，以确定哈希糟位置

Jdk1.7扩容的时候resize()使用的是头插法

hashMap对象丢失的原因有以下几点

  1. 并发赋值时被覆盖

  2. 己遍历区间新增元素会丢失。

  3. “新表”被覆盖。

  4. 迁移丢失。在迁移过程中,有并发时,next 被提前 null
